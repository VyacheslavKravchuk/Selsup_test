


# CrptApi.java - Java API для работы с "Честный знак"

Этот проект предоставляет Java класс `CrptApi` для взаимодействия с API системы "Честный знак". 
Класс обеспечивает thread-safe работу и ограничивает количество запросов к API для предотвращения перегрузки сервиса.

# CrptApi

`CrptApi` - это класс на Java, предназначенный для взаимодействия с API системы маркировки товаров Честный Знак.
 Он обеспечивает ограничение количества запросов в единицу времени для предотвращения перегрузки API. 
 Класс является thread-safe (потокобезопасным) и может использоваться в многопоточных приложениях.

## Обзор

Класс `CrptApi` предоставляет следующие возможности:

*   Ограничение количества запросов к API в заданный период времени
*   Запрос авторизации и получение пары "uuid - data"
*   Запрос аутентификации для получения токена (невозможен, потому как "data" - неподписанная УКЭПом строка)
*   Создание документа (имитация отправки, возвращает отправленный JSON)
*   Поддержка настройки URL API
*   Thread-safety для использования в многопоточных приложениях

## Конструкторы

Класс `CrptApi` имеет два конструктора:

1.  `CrptApi(TimeUnit timeUnit, int requestLimit)`:
    *   `timeUnit`: Единица времени для ограничения количества запросов (например, `TimeUnit.SECONDS`, `TimeUnit.MINUTES`).
    *   `requestLimit`: Максимальное количество запросов, разрешенных в течение `timeUnit`.
2.  `CrptApi(TimeUnit timeUnit, int requestLimit, String apiUrl)`:
    *   `timeUnit`: Единица времени для ограничения количества запросов.
    *   `requestLimit`: Максимальное количество запросов, разрешенных в течение `timeUnit`.
    *   `apiUrl`: URL API системы маркировки товаров (по умолчанию "https://ismp.crpt.ru").

## Параметры

*   `timeUnit`:  Единица измерения времени для лимита запросов.
*   `requestLimit`: Максимальное количество запросов в заданную единицу времени.
*   `semaphore`: `Semaphore` используется для контроля количества одновременно выполняемых запросов, обеспечивая thread-safety.
*   `httpClient`: `HttpClient` может использоватся для выполнения HTTP-запросов к API.
*   `apiUrl`: URL API, к которому будут отправляться запросы.
*   `gson`: `Gson` используется для сериализации и десериализации JSON.
*   `authToken`: (Имитация) Строка, представляющая токен аутентификации.  В реальной системе здесь должен быть токен, полученный после аутентификации с использованием ЭП.
*   `tokenExpirationTime`: Время истечения срока действия токена (не используется в текущей имитации).
*   `tokenLock`: Объект для синхронизации доступа к токену (не используется в текущей имитации).

## Методы

*   `createDocument(Document document, String signature)`:  Создает документ и отправляет его в API (имитация).  Перед отправкой запроса метод ожидает получения разрешения от семафора (`semaphore.acquire()`). После отправки (или в случае ошибки) метод освобождает разрешение (`semaphore.release()`).
    *   `document`: Объект `Document`, представляющий данные документа.
    *   `signature`:  Строка, представляющая подпись документа (в текущей версии используется имитация).
    *   Возвращает: void.  Выводит имитацию ответа сервера в консоль.
    *   Выбрасывает: `IOException`, `InterruptedException`.

*   `stringToBase64(String data)`: Преобразует строку в Base64 (не используется, потому как data - не подписана).
    *   `data`: Строка для преобразования.
    *   Возвращает: Строка в кодировке Base64.

*   `getAuthKey()`: Пользователь отправляет запрос авторизации и получает пары "uuid - data", 

    *   `uuid`: идентификатор текущей аутентификации
    *   `data`: строка на подпись пользователю

*   `convertToJson(Document document, String signature)`: Преобразует объект `Document` и подпись в JSON-строку.

## Модели данных (DTO)

*   `Document`:  Класс, представляющий документ для отправки в систему маркировки.  
*   `Содержит поля, такие как `description`, `doc_id`, `doc_status`, `products` и другие.
*   `Description`: Класс, содержащий описание документа (например, ИНН участника).
*   `Product`: Класс, представляющий продукт в документе.
*   `AuthKeyResponse`: Класс, представляющий ответ от сервера с идентификатором и строкой.

## Thread-Safety (Потокобезопасность)

Класс `CrptApi` разработан с учетом требований многопоточности:

*   Использование `Semaphore` для ограничения количества одновременных запросов.  Это предотвращает превышение лимита 
  запросов при одновременном использовании класса из нескольких потоков.
*   Метод `createDocument` использует `semaphore.acquire()` и `semaphore.release()` для управления доступом к API, 
  обеспечивая безопасную работу в многопоточной среде.
